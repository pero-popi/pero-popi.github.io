AWS 用語集

顧客のビジネス課題解決のために、ITシステムの設計や構成を提案し、システムを構築する役割を担うエンジニアのこと。

顧客のニーズを理解し、技術的な側面とビジネス的な側面の両方を考慮してソリューションを設計・提案し、実装や運用にも関与することがある。

AWSクラウドの設計を最適化するためのガイドライン、AWSが提供するクラウドサービスのベストプラクティス集。

運用上の優秀性、セキュリティ、信頼性、パフォーマンス効率、コスト最適化、持続可能性の6つの柱がある。

AWS アベイラビリティーゾーン

AWSリージョン内にある、互いに独立したデータセンター群。電力、ネットワーク、接続が冗長化されており、あるAZが故障しても他のAZでサービスを継続できるように設計されている。

AZは、広域災害や障害からの影響を最小限に抑えるために、物理的に離れた場所に配置されている。

パブリックサブネットには、インターネットゲートウェイへの経路が追加されているため、インターネットへの送受信が可能。

プライベートサブネットには、インターネットゲートウェイへの経路が追加されていないため、インターネットへの直接アクセスはできない。

AWSはクラウド環境のセキュリティを含む仮想化インフラストラクチャの保護に責任を負う。

他方、利用者は、自身のデータ、プラットフォームアクセスの管理、OSとアプリケーションなどのゲスト環境に対して責任がある。

AWS Virtual Private Cloud

AWS上で仮想的なネットワークを構築・制御するためのサービス。

パブリックなインターネットと分離された、独自のネットワーク環境を構築できる。

インターネットゲートウェイやNATゲートウェイを経由することなく安全にVPC内からデータにアクセスできる。VPCからAWSサービスに安全に接続するためのもの。

Amazon Virtual Private Network

AWS上で仮想プライベートネットワークを構築し、社内ネットワークや自宅など、どこからでもAWS上のリソースに安全にアクセスできるようにするサービス。

具体的には、オンプレミス環境とAWSのVPCを接続する (AWS Site-to-Site VPN) と、個々のクライアント端末からAWSにアクセスする (AWS Client VPN) の2種類がある。

Amazon Virtual Private Network CloudHub

AWS環境に接続する拠点が複数ある場合に、各拠点間のVPN接続を効率的に管理するためのAWSサービス。

各拠点とAWS間のSite-to-Site VPNを、VPN CloudHubを使用することで、安全な拠点間通信を確立できる。

AWS Identity and Access Management

AWSのリソースへのアクセスを安全に制御するためのサービス。

ユーザー、グループ、ロール、ポリシーを管理し、誰がどのリソースにアクセスできるかを細かく制御できる。

フルマネージド型インメモリキャッシュサービスで、アクセスの高速化のためのキャッシュレイヤー。

ウェブアプリケーションのパフォーマンス向上、データベースへの負荷軽減、高速なデータアクセスを実現する。

フルマネージドのインメモリキーバリューストアサービスで、Memcached互換のキャッシュエンジンを使用。

フルマネージドのインメモリキャッシュサービスで、Redis互換のキャッシュエンジンを使用。

フルマネージドNoSQLデータベースサービス。

自動スケーリング機能、高速で柔軟なデータベースとして、大規模なデータセットと高いスケーラビリティが求められるアプリケーション向け。

Amazon DynamoDB Accelerator

DynamoDBのインメモリキャッシングサービス。

DAXを使用すると、DynamoDBへの読み取り要求を高速化できる。

Amazon Simple Storage Service

オブジェクトストレージサービス。ファイルや画像をインターネットに保存したり、バックアップやアーカイブに利用したり、ウェブサイトの静的コンテンツをホストしたりするなど、様々な用途で利用。

低コストで利用でき、高い耐久性・可用性があり、無制限のストレージ容量が利用できる。

Amazon S3を基盤としたデータレイク。データレイクとは、構造化データ、非構造化データなど、あらゆる形式のデータを一元的に保存・管理するリポジトリのこと。

S3をデータレイクとして活用することで、大量のデータをコスト効率よく保存し、様々な分析や機械学習に活用できる。

オンプレミス環境のファイルサーバーをAmazon S3に接続し、ファイル共有 (SMB/NFS) を介してS3にデータを保存できるようにするサービス。

AWSのS3バケットへのデータ転送を高速化するための機能。

特に、遠隔地からS3にデータをアップロードする際に、CloudFrontのエッジロケーションを活用することで、転送速度を向上できる。

リレーショナルデータベースサービスで、MySQLおよびPostgreSQLと互換性のあるデータベースエンジン。

商用データベースの性能とオープンソースデータベースのコスト効率を兼ね備え、クラウドに最適化された設計。

Amazon Auroraデータベースの読み取り専用のレプリカ。

Amazon Aurora互換のリレーショナルデータベースで、自動的にスケーリングする完全マネージド型のサービス。

ワークロードの変動に合わせて自動的にクラスターのキャパシティを拡張・縮小するため、トラフィックの増減に柔軟に対応できる。

Amazon Elastic Block Store

ブロックストレージサービス。EC2インスタンスにアタッチして、仮想ディスクとして利用できるストレージ。

EC2に永続的なストレージを提供することで、インスタンスが停止してもデータを保持できる。

Amazon Elastic Compute Cloud

AWSのクラウド上に仮想サーバーを構築できるサービス。

仮想サーバー構築サービスであり、AWS上でWebアプリケーションやデータベースなどを動かすための基盤を提供する。

Amazon Machine Image

EC2インスタンスを起動する際に使用されるテンプレート。

OSやアプリケーション、設定、データなどが含まれており、EC2インスタンスの迅速なデプロイやバックアップに役立つ。

デディケイテッド ホスツ

専用ホスト。ソフトウェアライセンスの利用を維持しながらクラウドに移行するのに最適なソリューション。

ホスト単位でサーバーリソースを完全に専有できる仮想マシン環境を提供する。

フリートは、1つのAPIで複数のインスタンスを起動、管理し、コスト最適化やキャパシティの柔軟な調整を可能にする機能。

オンデマンドインスタンス、リザーブドインスタンス、スポットインスタンスなど、異なるタイプのEC2インスタンスを組み合わせて構成されたグループ。

EC2の揮発性ブロックストレージ。

EBSのようにデータが永続化されない一方で、EBSより高IOPS、高スループット、低レイテンシーのため、高いI/Oパフォーマンスが必要な一時ストレージに最適。

AWSのEC2で提供される、スポットインスタンスをまとめたグループのこと。

スポットインスタンスは、AWSが空き資源を利用して安く提供するインスタンスタイプで、スポットフリートは、そのスポットインスタンスを効率的に利用するための仕組み。

指定したアベイラビリティーゾーンとAWSリージョンでEC2インスタンスの容量を確保できる。

サーバーを管理することなく、イベント発生時にコードを実行できるサーバーレスコンピューティングを実現する、FaaS（Function as a Service）型のサービス。

Amazon Simple Queue Service

完全マネージド型のメッセージキューイングサービス。自動スケーリング。

アプリケーション間でメッセージを非同期にやり取りするための仕組みを提供し、マイクロサービスのような分散システムを構築する際に役立つ。

フルマネージド型データウェアハウスサービス。

高速にデータ分析できるのが特徴で、大規模なデータを効率的に処理する。

スペクトラム

Amazon RedshiftからAmazon S3に保存されたデータを、データをRedshiftにロードすることなく直接クエリできる機能。

AWS Elastic Load Balancing

ロードバランシングサービス。アプリケーションへのトラフィックを複数のターゲット (EC2 インスタンス、コンテナなど) に分散することで、サービスの可用性を向上させ、負荷を均一化する。

ELBは、ALB (Application Load Balancer)、NLB (Network Load Balancer)、GLB (Gateway Load Balancer) の3種類があり、それぞれ異なるユースケースに対応している。

AWS Application Load Balancer

HTTP/HTTPSトラフィックを分析し、リクエストの内容 (URLやヘッダーなど) に基づいて適切なサーバーに振り分けることで、柔軟な負荷分散を実現する。

ホストベースのルーティング機能を備えているため、異なるURLを持つ異なるターゲットグループにリクエストを転送できる。

AWS Network Load Balancer

アプリケーションへのトラフィックを分散するロードバランサーの一種で、特に高パフォーマンスと低遅延が求められる場合に利用される。

TCP/UDPのトラフィック分散に最適。

AWS Gateway Load Balancer

ロードバランサーの一種で、ファイアウォールや侵入検知システムなどのサードパーティ製仮想アプライアンスを、デプロイ、スケーリング、管理を容易にするサービス。

バックエンドサーバーの登録解除や削除を行う際に、現在進行中の接続 (コネクション) を切断せずに、処理が完了するまで一定時間待つ機能。これにより、アプリケーションに急な中断が発生するのを避け、ユーザーへの影響を最小限に抑えることができる。

ロードバランサーがアプリケーションサーバーを使用不可能と判断した場合に発生する状況を処理するために使用される。

リクエストを複数のアベイラビリティーゾーンに分散する。

特定のクライアントからのリクエストが同じサーバーインスタンスに転送されるように設定するもの。

システムの需要に基づいてコンピューティング・リソースを自動的に割り当てるクラウド・コンピューティング機能。

Auto Scalingを利用することで、CPU使用率に応じてインスタンスの数を自動的に増減させることができる。これにより、CPU使用率をなるべく一定に保つことが可能。

自動スケーリングと管理の目的で論理グループとして扱われるインスタンスの集合が含まれる。

複数のデータベースサーバが連携して一つのデータベースとして動作する仕組み。

コンピューターやネットワーク機器が一定時間あたりに転送できるデータ量のこと。

基本的にスループットの数値が高ければ高いほど、1秒あたりに転送できるデータ量が増え、通信速度が速くなる。

Network Address Translation Gateway

プライベートなIPアドレスを持つリソースがインターネットにアクセスするために使用されるネットワークアドレス変換サービス。

プライベートなIPアドレスをパブリックなIPアドレスに変換し、インターネットとの通信を可能にする。これにより、プライベートネットワーク内のリソースが直接インターネットに公開されることなく、外部サービスにアクセスできるようになる。

Amazon Relational Database Service

リレーショナルデータベースサービス。クラウド環境でリレーショナルデータベースを簡単に設定、運用、スケールできるウェブサービス。

MySQL、PostgreSQL、Oracle、SQL ServerなどのRDBMSを、手軽にクラウド上で利用することができる。

データベース接続の管理を最適化し、接続プールを利用することでアプリケーションのスケーラビリティを向上させるマネージドプロキシサービス。

AWS Command Line Interface

AWSのサービスをコマンドラインから操作するための、オープンソースのツール。

コマンドを入力することで、AWSマネジメントコンソールで行う操作と同様のことが行える。また、スクリプト化することで、AWS環境の操作を自動化することも可能。

仮想プライベートサーバーサービス。

AWSのクラウド上に、ウェブサイトやウェブアプリケーションなどのホスティングに必要なリソース (CPU、メモリ、ストレージ、データ転送) をパッケージ化して、手軽に利用できるサービス。初心者でも簡単にサーバーを立ち上げ、ウェブサイトやアプリケーションを構築することが可能。

デフォルトは、すべてのインバウンドトラフィックは拒否され、すべてのアウトバウンドトラフィックは許可される。

ステートフルで、一度インバウンドを許可するとアウトバウンドも自動的に許可されるAllow‐onlyモデル。

Network Access Control List

ネットワークトラフィックの制御を行うためのリストで、サブネットレベルでインバウンドとアウトバウンドのトラフィックを許可または拒否する。

ACLはVPC (仮想プライベートクラウド) 内のサブネットに対して設定され、そのサブネットの境界でトラフィックをフィルタリングする。

インバウンド (ネットワーク外部からサブネット内へのトラフィック) とアウトバウンド (サブネット内からネットワーク外部へのトラフィック) をそれぞれ個別に制御できる。

AWS Web Application Firewall

AWS WAFで利用される、Webアプリケーションへのアクセスを制御するためのリスト。

Web ACLは、ルールをグループ化し、特定の条件に一致するリクエストを許可または拒否することで、Webアプリケーションを保護する。

Web Access Control List

Webアプリケーションファイアウォールで、WebサイトやWebアプリケーションをサイバー攻撃から保護するセキュリティサービス。

具体的には、SQLインジェクションやクロスサイトスクリプティング (XSS) などの攻撃を検知・遮断し、Webアプリケーションの脆弱性を悪用した攻撃から保護する。

オンプレミス環境とAWSクラウドの間を、インターネットを経由せずに専用のネットワーク回線で接続するサービス。

高帯域幅、プライベート接続、安定した接続が可能となり、オンプレミスとクラウド間の大量データ転送などに適している。

フルマネージド型サービスで、APIの作成、公開、管理、監視、セキュリティを効率的に行うためのツール。

REST/HTTP、WebSocket APIをサポートし、認証・承認、トラフィック管理、モニタリング、バージョン管理といったAPIのライフサイクル全般を管理できる。

スケジュールされたルールをLambda関数にマッピングできるフルマネージドのイベントバスサービス。

Amazon VPCのリソースとオンプレミスネットワークの間のルーティングを容易にするためのAWSマネージドサービス

複数のVPCやオンプレミスネットワークを相互に接続するためのハブのような機能を提供する。

等コストマルチパスルーティングを使用すると、同じ宛先へのトラフィックを複数の等コストパスに分散でき、VPNトンネルを追加することで、全体的なスループットを向上させることができる。

AWS内のデータの抽出、変換、転送を自動化するための管理サービス。DynamoDBのデータをS3へエクスポートしたり、S3のデータをDynamoDBへインポートできる。

このサービスを使えば、指定した順序でデータのムーブメントとデータ処理アクティビティを実行できる。

ITインフラ設備やシステムリソースを、需要に合わせて準備、割り当て、設定、管理するプロセスを指す。

具体的には、サーバー、ネットワークデバイス、アプリケーション、ユーザーアカウントなどを、ユーザーが利用できる状態にするために行う作業。

ITサービスを運用する上で必要なハードウェア、ソフトウェア、ネットワークなど。

AWSクラウドでリソースを簡単にプロビジョニング&管理できるサービス。

AWSのリソース (サーバー、データベース、ネットワークなど) をコードで定義し、自動的に作成、更新、削除するサービス。

テンプレートと呼ばれるJSONまたはYAML形式のテキストファイルでAWSリソースの構成を記述し、そのテンプレートに基づいてスタックと呼ばれるリソース群を作成する。

単一のCloudFormationテンプレートを複数のAWSアカウントおよびリージョンにデプロイできる。これにより、企業全体で一貫したリソースプロビジョニングプロセスを実現できる。

AWSで提供される静的なパブリックIPv4アドレス。

EC2インスタンスやネットワークインターフェイスに割り当てると、インスタンスを停止・起動してもIPアドレスが変わらず、固定したIPアドレスでインターネットにアクセスできるようになる。

Amazon Elastic File System

AWSが提供するフルマネージド型のファイルストレージサービス。

AWSサービス (特にEC2インスタンス) 間でファイルを共有するために利用される。リージョン内のすべてのAvailability Zone (AZ) にわたって自動的に冗長化される。

複数のEC2インスタンスから同時にマウントできる。File Systemサイズは自動的に拡張される。

Amazon Elastic Container Service

AWSのマネージド型のコンテナオーケストレーションサービス。

コンテナ化されたアプリケーションを簡単にデプロイ、管理、スケーリングできる。

Amazon ECSやAmazon EKSでコンテナを実行するためのサーバーレスコンピューティングエンジン。

コンテナを起動するために、サーバーやクラスターをプロビジョニング、設定、スケーリングする必要がなく、従量制料金でコンテナを動かすことができる。

サーバーレスのコンテナ実行環境。Fargateを利用することで、インフラ (サーバーやEC2インスタンス) の管理を意識することなく、コンテナをデプロイし実行できる。

システム障害が発生した場合に、自動的に予備システムに切り替える仕組み。

DNSサービス。AWSの他のサービスとの連携が簡単で、Amazon S3やELB (Elastic Load Balancing) などのIPが動的に変更されるサービスを利用する上でも、Amazon Route 53を使ったDNSの設定が役立つ。

フェイルオーバーオプションを利用することで、Webサイトに障害が発生した場合に静的なS3Webサイトへ自動的にフェイルオーバーさせることができる。

Webアプリケーションなどでユーザの認証 (ログイン、ログアウト) などの機能を実装できるサービス。

UI上で簡単な設定をするだけでユーザ管理を実装できる。

組み込みのユーザー管理とサインアップ/サインイン機能を提供するマネージド型サービス。

API Gatewayと統合することで、APIアクセスの認証や認可を行うことができる。

外部IDプロバイダーを使用してユーザーを認証する、AWSのリソースへのアクセスを許可するために、ユーザーに一時的なAWS認証情報を発行するサービス。

主に、アプリケーションユーザーを認証済みのAWSリソースに接続するために使用される。

API Gatewayに設定することで、Amazon Cognitoがトークンの有効性を検証し、適切なアクセス権を持つユーザーのみがリソースにアクセスできるようになる。

なるべくユーザーに近いネットワークでのアクセスを提供するためのサービスで、高速で安定したアクセスを実現させるサービス。

リージョンやAZは配置されている国や地域が限定されているため、サービス利用者が直接これらのサーバーにアクセスしようとした場合、物理的な距離が遠い場合がある。

静的、動的なWebコンテンツを高速かつ安全に配信するための、AWSが提供するコンテンツデリバリネットワーク (CDN) サービス。

世界中に配置されたエッジロケーションを活用することで、エンドユーザーに近い場所からコンテンツを配信し、低レイテンシー (遅延時間) を実現する。

Amazon S3バケットとALBの両方をオリジンとしてサポートする。

データ転送における指標のひとつで、転送要求を出してから実際にデータが送られてくるまでに生じる、通信の遅延時間のこと。

この遅延時間が短いことをレイテンシが小さい (低い) 、遅延時間が長いことをレイテンシが大きい (高い) と表現する。

ソフトウェアライセンスの使用状況を追跡し、管理することで、コストの最適化とコンプライアンスの維持を支援するサービス。

AWS Application Discovery Service

企業がオンプレミスのデータセンターからAWSへの移行を計画する際に、アプリケーションの依存関係を識別し、移行計画を支援するために使用。

オンプレミス環境のサーバーやデータベースの設定、使用状況、パフォーマンスなどの情報を収集・分析し、AWSクラウドへの移行を計画するのに役立つサービス。

Virtual Interface (仮想インターフェース)

プライベートVIF: パブリックIPアドレスで接続。

トランジットVIF: Direct Connect Gateway経由でTransit Gatewayに接続。

コンピューティングリソース (EC2インスタンス、オートスケーリンググループ、EBSボリューム、Lambda関数など) の選定と配置に関する推奨事項を提供するサービス。これにより、パフォーマンスを向上させつつコストを削減することができる。

広範なデスクトップブラウザと実際のモバイルデバイスでテストすることにより、ウェブアプリケーションとモバイルアプリの品質を改善できるようにするアプリケーションテストサービス。

AWSのインフラストラクチャ、サービス、API、ツールをオンプレミス環境に拡張できるフルマネージドサービス。

SOC、ISO、PCIなどの認定証明書や監査報告書をセルフサービスでダウンロードできるポータルを提供し、契約文書 (AWS Business Associate Addendum など) の管理も行えるサービス。

AWSアカウント内のリソースがコンプライアンス違反していないかを継続的にチェックするサービス。

複数のハードディスクドライブを組み合わせて、1つのディスクアレイ (ディスクグループ) として扱う構成のこと。これにより、データの冗長性やパフォーマンスの向上を図る。

RAID 0 (ストライピング) により、複数のディスクドライブにデータを分散して並列書き込みや読み込みを行うことで、I/O性能が向上。

RAID 1 (ミラーリング) はデータの冗長性を提供するが、読み取りに対してのみ性能が向上する。

GraphQLを活用し、複数のデータソースを統合し、安全にアクセス・操作できるAWSのサーバーレス型マネージドサービス。

リアルタイム機能やデータの同期、必要なデータのみの取得など、多様な機能を備える。
